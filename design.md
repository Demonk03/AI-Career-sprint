# AI Career Coach — Design Doc
_Дата: 2026-03-29_

## Проблема

Люди, застрявшие на карьерном распутье, не могут самостоятельно пройти структурированную диагностику. Без фасилитатора методика не работает — человек уходит в сторону, не докапывается до сути, не делает выводов.

## Аудитория MVP

IT-специалисты (джуны и мидлы), которые застряли на текущей позиции и не понимают, куда двигаться.

## Решение

Веб-приложение: пользователь проходит структурированную форму (Сессия 1 карьерной методики), AI синтезирует ответы и возвращает персональный карьерный отчёт.

---

## UX-флоу

### 1. Лендинг
- Одно предложение — что это и для кого
- Кнопка "Начать диагностику"

### 2. Многошаговая форма (4 блока, прогресс-бар)

**Шаг 1 — Цели**
- Куда хочешь прийти через 6–9 месяцев?
- Почему это важно для тебя?
- Подсказка: не думай о "правильном" ответе, пиши первое что приходит

**Шаг 2 — Ресурсы**
- Что делаешь стабильно хорошо? (компетенции)
- Какой опыт уже есть, которым можно гордиться?
- Как тебя воспринимают коллеги?
- Подсказка: представь что рассказываешь об этом другу, не на собесе

**Шаг 3 — Личная эффективность**
- Когда ты максимально продуктивен? Что это за условия?
- Что точно не работает — задачи, форматы, обстановка?
- Что выбивает из состояния чаще всего?

**Шаг 4 — SWOT**
- Сильные стороны (что помогает двигаться)
- Слабые стороны / зоны роста (чего не хватает)
- Возможности (что открывается из сильных сторон)
- Угрозы (что может помешать из-за слабых сторон)
- Подсказка: SWOT — это не про идеальный образ, а про честную картину

### 3. Обработка
- Лоадер с текстом "Анализируем ваши ответы..."
- POST-запрос на бэкенд → OpenAI API → результат

### 4. Страница результата
Структура отчёта:
- **Точка 0** — краткое описание текущего состояния (AI формулирует из ответов)
- **Сильные стороны** — что стоит опираться
- **Пробелы** — зоны роста, которые важны для цели
- **Рекомендации** — 3–5 конкретных шагов
- **Скачать результат** — кнопка (markdown или PDF)

---

## Технический стек

| Компонент | Решение |
|---|---|
| Бэкенд | Python + FastAPI |
| Фронтенд | HTML + CSS (без фреймворков) |
| AI | OpenAI API (GPT-4o) |
| База данных | Нет (MVP — данные живут в сессии) |
| Хостинг | Railway или Render |

---

## Что не входит в MVP

- Аккаунты и сохранение прогресса между визитами
- Сессии 2 и 3
- Чат-интерфейс с AI в реальном времени
- Авторизация

---

## Следующая итерация (v2)

- Аккаунты + сохранение сессий
- Сессия 2: таблица роль-функционал с оценками
- Сессия 3: план действий 70/20/10
- Чат-режим внутри каждого блока (AI задаёт уточняющие вопросы)
