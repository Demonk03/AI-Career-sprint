# AI Career Coach — контекст проекта

## Что это

Веб-приложение для карьерной диагностики IT-специалистов (джуны и мидлы). Пользователь проходит 3 сессии — AI синтезирует ответы и возвращает персональный анализ и план развития. Не чат, не диалог — структурированные формы и отчёты.

Цель: портфолио-проект, демонстрирующий продуктовое мышление (проблема → MVP → обратная связь → итерации).

## Стек

| Компонент | Решение |
|---|---|
| Бэкенд | Python + FastAPI |
| Фронтенд | HTML + CSS (без фреймворков), Jinja2-шаблоны |
| AI | OpenAI API (GPT-4o) |
| Хранение данных | localStorage в браузере (без БД) |
| Фидбек | Telegram Bot → отдельный чат |
| Деплой | Render (render.yaml в корне) |
| Репо | https://github.com/Demonk03/AI-Career-sprint |

## Структура файлов

```
main.py               — FastAPI-приложение, все роуты
prompts.py            — промпты для всех трёх сессий
templates/
  index_minimal.html  — лендинг
  survey.html         — сессия 1: форма (4 блока)
  result.html         — результат сессии 1
  session2.html       — сессия 2: таблица задач
  result2.html        — результат сессии 2
  session3.html       — сессия 3: доп. вопросы + генерация плана
  result3.html        — результат сессии 3 (план 70/20/10)
static/style.css      — базовые стили
render.yaml           — конфиг деплоя
.env                  — переменные окружения (не в git)
```

## UX-флоу

```
Лендинг → Сессия 1 → Результат 1 → Сессия 2 → Результат 2 → Сессия 3 → Результат 3
```

Данные между сессиями передаются через localStorage:
- `session1Data` — ответы сессии 1
- `session2Tasks` — таблица задач сессии 2
- `careerReport` — отчёт сессии 1
- `session2Report` — отчёт сессии 2
- `session3Report` — отчёт сессии 3

## Сессии

### Сессия 1 — Диагностика (`/survey` → `/result`)
4 блока: Цели / Ресурсы / Личная эффективность / SWOT.
Промпт: `build_prompt()` в prompts.py.
Отчёт: Точка 0, Сильные стороны, Пробелы, Рекомендации.

### Сессия 2 — Функционал (`/session2` → `/result2`)
Динамическая таблица: задача + энергия (Даёт/Нейтрально/Выматывает) + % времени.
Валидация: каждое поле обязательно, % не > 100, сумма % не > 100.
Промпт: `build_prompt_session2()` — получает контекст сессии 1.
Отчёт: Картина роли, Энергетический баланс, Соответствие цели, Рекомендации.

### Сессия 3 — План 70/20/10 (`/session3` → `/result3`)
2 вопроса: часы в неделю на развитие + ограничения (необязательно).
Промпт: `build_prompt_session3()` — получает контекст сессий 1 и 2.
Отчёт: Фокус, 70% (текущая работа), 20% (смежный опыт), 10% (обучение), первые 2 недели.

## Фидбек

После каждой сессии — блок с 5 звёздами и опциональным комментарием.
POST `/feedback` → Telegram Bot → чат фидбека.
Формат сообщения: `Оценка сессии N: ★★★★☆ (4/5)\n\nКомментарий`.

## Скачивание

Каждая страница результата имеет кнопку "Скачать .md".
В файл входят: ответы пользователя + отчёт GPT.

## Переменные окружения

- `OPENAI_API_KEY` — ключ OpenAI
- `TELEGRAM_BOT_TOKEN` — токен бота для фидбека
- `TELEGRAM_CHAT_ID` — ID чата для получения оценок
